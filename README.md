# Аналіз екологічної та радіаційної обстановки в зоні розташування АЕС України (2018–2025)

### Мета проекту

Дослідити динаміку радіоактивних викидів та скидів українських атомних електростанцій. Основна мета — виявити приховані закономірності в даних моніторингу, перевірити відповідність показників нормам безпеки та побудувати модель машинного навчання для прогнозування або класифікації екологічного стану в зонах розташування АЕС.

### Джерело даних

Дані отримані з Єдиного державного порталу відкритих даних:

* **Набір даних:** [Екологічна та радіаційна обстановка в зоні розташування атомних електростанцій](https://data.gov.ua/dataset/4a9d3d56-bd95-4c3e-97e7-1cdc7bcbd445)
* **Розпорядник:** Міністерство енергетики України

---

### Питання та гіпотези для аналізу

#### 1. Пошук аномальних відхилень у викидах

* **Питання:** Чи існують у часових рядах точки (квартали), коли рівень радіоактивних викидів був нетипово високим для конкретної АЕС?
* **Гіпотеза:** Показники можуть виходити за межі "нормального" шуму через технічні збої або особливості експлуатації, навіть якщо вони не перевищують гранично допустимі державні норми.

#### 2. Ідентифікація АЕС за «цифровим відбитком»

* **Питання:** Чи можна за хімічним складом викидів (співвідношенням $Cs^{137}$, $Co^{60}$ та йоду) точно визначити станцію без знання її назви?
* **Гіпотеза:** Кожна станція має унікальний профіль викидів, зумовлений типом реакторів (ВВЕР-440 vs ВВЕР-1000), віком обладнання та специфікою систем очистки.

#### 3. Факторний аналіз сумарного індексу викидів

* **Питання:** Які саме типи викидів (інертні гази, цезій чи йод) мають критичний вплив на формування загального індексу викидів (`index_radioactive_releas`)?
* **Гіпотеза:** Загальний екологічний ризик станції визначається не всіма показниками рівномірно, а домінуванням одного-двох специфічних факторів.

### Стек технологій

* **Python** (Pandas, NumPy) — для очищення та підготовки даних.
* **Matplotlib / Seaborn** — для візуалізації трендів та кореляцій.

### Структура проєкту
Проєкт реалізовано як багатомодульну систему в Docker-середовищі:

```text
.
├── core/                       # Спільна логіка (менеджер БД)
├── data/                       # Вхідні дані
│   ├── raw/                    # Вихідний Excel-файл
│   └── test/                   # Дані для тестування
├── data_load/                  # Модуль ETL (завантаження в БД)
├── data_quality_analysis/      # Перевірка якості даних
├── data_research/              # ML-дослідження та статистика
├── db/                         # Сховище SQLite (Shared Volume)
├── notebooks/                  # Експерименти (EDA)
├── reports/                    # Результати (Shared Volume)
│   ├── figures/                # Графіки (PNG)
│   └── files/                  # Звіти (JSON)
├── src/                        # Вихідний код компонентів
├── tests/                      # Юніт-тести
├── visualization/              # Модуль генерації графіків
├── web/                        # Веб-інтерфейс (Streamlit)
├── docker-compose.yml          # Конфігурація розробки
└── docker-compose.prod.yml     # Production-конфігурація
```

### Опис сервісів та обмін даними
Система працює за принципом конвеєра (Pipeline), де результати передаються за допомогою **комбінованого підходу**:

1.  **data_load**: Зчитує XLSX, створює таблиці та імпортує дані в SQLite.
    * *Передача:* Записує файл у спільний том `./db`.
2.  **data_quality_analysis**: Аналізує повноту даних та дублікати.
    * *Отримання:* Читає БД з `./db`.
    * *Передача:* Записує JSON у спільний том `./reports/files`.
3.  **data_research**: Виконує статистичний аналіз та ML (пошук аномалій).
    * *Отримання:* Читає БД з `./db`.
    * *Передача:* Записує JSON у спільний том `./reports/files`.
4.  **visualization**: Будує графіки на основі ML-результатів.
    * *Отримання:* Читає JSON-звіти.
    * *Передача:* Записує PNG у спільний том `./reports/figures`.
5.  **web**: Відображає фінальні результати.
    * *Отримання:* Читає JSON та PNG з тому `./reports`.

### Команди запуску

#### 1. Підготовка
Переконайтеся, що файл даних лежить у папці:
`./data/raw/nuclear_safety_q4_2025.xlsx`

#### 2. Запуск усього проєкту
Для автоматичного складання образів та послідовного запуску всіх модулів виконайте:
```bash
docker compose up --build
```

#### 3. Перевірка результатів
Після завершення роботи розрахункових модулів (статус `Exited 0`), відкрийте браузер:
* **URL:** `http://localhost:8501`

#### 4. Зупинка та очищення
```bash
docker compose down -v
```

---

### Мережі та Порти
* **Мережа:** `nuclear_network` (bridge) — забезпечує ізоляцію сервісів.
* **Порт:** `8501` — порт веб-інтерфейсу Streamlit, прокинутий на хостову машину.
